#+title: org-bootstrap-publish — Changelog
#+author: James Dyer
#+options: toc:nil

All notable changes to this package.  Dates follow the commit date
(ISO 8601).

* Unreleased

Pending work on =main= not yet covered by a release tag:

- Multi-source sites.  =org-bootstrap-publish-source-files= (plural) takes a list of Org files and merges their headings into one site.  Used for an umbrella build that aggregates per-topic blogs (art, linux, emacs, etc.) into one navigation tree.  Save-hook rebuild matches whichever of the configured files is being saved, so incremental rebuilds keep working across the set.  When both are set, =-source-files= takes precedence over =-source-file=.
- Sidebar arbitrary-URL links.  =org-bootstrap-publish-menu-links= renders an alist of =(label . url)= pairs after the existing =-menu-tags=.  Aggregator sites generally want links to section landings (=/blog/=, =/photos/=) and specific posts rather than tag pages, which =-menu-tags= can't express.
- Section landing pages.  Each distinct =EXPORT_HUGO_SECTION= root gets a =/<root>/index.html= listing every post in that section (and sub-sections).  Skipped when a post with =EXPORT_FILE_NAME: index= already owns the URL.  Without this the sidebar's =/blog/= or =/photos/= links 404 unless every section has a hand-authored landing post.  Pages paginate at =org-bootstrap-publish-posts-per-page= just like the home index, with =/<root>/page/N/= for the second page onwards.  =-menu-links= entries can opt the destination into a =:style list= flat bullet list (defaults to =cards=, the same Bootstrap card grid the home page uses); useful for blog-shaped sections where thumbnails are noise.
- Cross-build HTML cache.  =org-bootstrap-publish-cache-dir= (defaults to =$XDG_CACHE_HOME/org-bootstrap-publish/=) holds rendered =ox-html= output keyed by a sha256 of the input.  Each build hits the cache for unchanged post bodies and skips =ox-html= entirely; stale entries are swept at the end of each build.  Cache lives outside =output-dir= so it does not get deployed.  Set the var to nil to disable.
- Per-build memoization of post cards and atom feed entries.  A single post object yielded the same =--card= and =<entry>= block every time it appeared, but the renderers were re-running for every section page and every per-tag feed.  Now a post-keyed hash table caches the result for the duration of the build.  Combined with the cross-build HTML cache this took the dyerdwelling site (1200 posts, 6 source files, 576 tag feeds) from a 53s cold rebuild down to a 33s warm rebuild (~38% faster); section pages alone fell from 6.8s to 0.7s.
- =-menu-links= is now the canonical sidebar variable; =-menu-tags= is deprecated and gets translated into the merged list (preserving its historical bullet-list rendering).  The =:style= plist key now applies to tag pages too — use =:style cards= on a =/tags/foo/= entry to render that tag's page as a Bootstrap card grid instead of the default flat list.  Tag and section pages share the same lookup, so one variable covers all the navigation styling.
- =org-bootstrap-publish-serve= now accepts the multi-source list.  Previously it errored with "Set =org-bootstrap-publish-source-file= first" when only =-source-files= was set, and only installed the rebuild-on-save hook on a single buffer; now it walks the configured list and adds the hook to every source file's buffer up front.
- Gallery posts.  A heading marked =:EXPORT_HUGO_TYPE: gallery= renders a Bootstrap masonry grid of every image found in =static/<section>/= relative to the source file, sorted by filename descending so date-prefixed files come out newest-first.  The heading body still renders above the grid, so the existing intro paragraph survives.  Each tile links to the full-size image.  Driven by =org-bootstrap-publish--render-gallery=; the dispatch in =--write-post= keeps non-gallery posts on the standard renderer.
- =EXPORT_FILE_NAME: index= now collapses the post URL to =/<section>/= rather than =/<section>/index/=, matching Hugo's content-bundle convention.  Lets a single heading own a section's landing page, which is how the new gallery posts are wired.
- =org-bootstrap-publish--copy-static= mkdirs the destination's parent before each =copy-directory=, so nested entries in =org-bootstrap-publish-static-dirs= (e.g. =static/art--gallery=) work without any extra setup.  Useful for siblings sharing one source tree where each site only wants part of =static/=.
- Hugo-shortcode preprocessor.  Recognises =\{\{< youtube ID >\}\}=, =\{\{< video src="..." >\}\}=, and =\{\{< figure src="..." caption="..." >\}\}= whether they sit on their own or inside a =#+begin_export md= / =#+end_export= wrapper, and rewrites them to a =#+begin_export html= block emitting a Bootstrap iframe / =<video>= / =<figure>=.  Without this, ox-html silently dropped the md export blocks and the embeds vanished.  =/foo= paths in =video= / =figure= shortcodes are rewritten to =<site-path>static/foo= so Hugo's flattened-static convention keeps working.
- Suppress =org-confirm-babel-evaluate= / =org-export-use-babel= inside the package's export so editing a README with src blocks no longer produces "Evaluate this emacs-lisp code block on your system?"  prompts on every save.
- Light/dark theme toggle.  Content-area colours move into CSS variables with a =[data-bs-theme="dark"]= override; an inline =<head>= script applies the saved theme (or =prefers-color-scheme=) before first paint; a sidebar button flips state and persists to =localStorage=.  Button label always names the currently-active mode.
- Save-hook rebuild now runs in a "fast" mode that skips per-tag HTML, per-tag feeds, and the site feed.  These embed fully exported post bodies via =ox-html= and were adding ~10 s per save; fast-mode rebuilds complete in ~0.5 s.  The skipped listings refresh on the next full build (=M-x org-bootstrap-publish-async=).
- Save-hook rebuild also re-copies =assets/=, so stylesheet edits flow through to =public/= without needing a separate full build.
- Live-reload for the dev server.  Each build writes a timestamp to =public/reload-token=; every generated page embeds a small script that polls that file once a second from =localhost= and calls =location.reload()= when it changes.  The hostname check means the script is a no-op on the deployed site.
- Layout choice.  =org-bootstrap-publish-layout= (=sidebar= by default; =rightbar= for the same vertical bar on the right edge; =topbar= for a horizontal nav above the content) selects the page chrome.  The wrapper div carries a =site-layout-<name>= class so additional layouts can be added via user CSS without changing the generator.
- Disqus comment threads.  =org-bootstrap-publish-disqus-shortname= (string, nil disables) injects a =<div id="disqus_thread">= plus the standard embed script at the bottom of every post and gallery page.  The injected script skips =localhost=/=127.0.0.1= so the dev server doesn't create stray threads under your shortname.  Set per-site (e.g. one shortname for the umbrella site, another for a sub-site) to keep threads partitioned.
- User-extensible shortcode registry.  =org-bootstrap-publish-shortcodes= is an alist of =(NAME . FUNCTION)= entries that extends the built-in =youtube=/=video=/=figure= preprocessor to arbitrary Hugo-style shortcodes.  The function receives a plist of parsed =key="value"= args (or nil for argless forms) and returns raw HTML, which gets wrapped in a =#+begin_export html= block before ox-html runs.  Lets per-site configs handle one-off Hugo shortcodes (interactive widgets, build-time directory listings, third-party iframes) without forking the package.  Note: handlers must be =fboundp= in whichever process renders the build, so the synchronous build path picks them up from your init/script automatically; the async batch path does not load your init, so async builds will skip rendering for any handler not defined inside the package itself.
- =--rewrite-static-src= now also normalises =file://...path/static/X= URLs in the exported HTML to =<site-path>static/X=.  ox-html turns absolute file links like =[[~/foo/static/bar.jpg]]= into =<img src="file:///home/.../static/bar.jpg">=, which 404 once deployed; this rewrite catches any =file://= URL containing a =/static/= segment so legacy DCIM-style image links keep working without editing the org source.
- Per-site theme overrides.  =org-bootstrap-publish-theme-overrides= is an alist of =(PROPERTY . VALUE)= pairs that emits an inline =<style>= block overriding the stylesheet's CSS custom properties (=--obp-sidebar-bg=, =--obp-link=, etc.).  The override is applied against =:root= /and/ every =[data-obp-theme=...]= selector, so it survives the light/dark/emacs toggle -- useful for locking site-wide branding (e.g. a fixed sidebar colour) without forking =assets/style.css=.  Property names accept the leading =--= or omit it.
- Optional body background image.  =org-bootstrap-publish-background-image= takes a URL or site-relative path and emits a =body { background-image: url(...); background-size: cover; background-attachment: fixed; }= rule into the page head.  nil leaves the active theme's =--obp-body-bg= colour showing through unchanged.  When set, the helper also emits a =.content-inner= rule giving the post column a solid =--obp-body-bg= backing (with modest padding and a 6px radius) so titles and body text stay readable over busy banner images; override =--obp-body-bg= via =-theme-overrides= (e.g. to =rgba(...)=) for translucency.
- Optional background-image blur.  =org-bootstrap-publish-background-blur= (integer px, nil for none) softens busy backgrounds without touching foreground text.  Implemented as =filter: blur(Npx)= on the fixed =body::before= overlay that carries the image, with =transform: scale(1.05)= so the blur's feathered edge does not leak past the viewport.
- Optional background-image opacity.  =org-bootstrap-publish-background-opacity= (number 0.0--1.0, nil for opaque) fades the image toward the active theme's =--obp-body-bg= colour for a subtler wash; combine with =-blur= for a heavily-defocused background hint.  Both knobs ride on the same =body::before= overlay, which is now used for every body background regardless of blur/opacity, so the rendering is consistent across all configurations.

* 2026-04-23

** Added

- Initial package: build a Bootstrap 5 static site from a single Org-mode file, one top-level heading per post.  Includes index with responsive post cards, per-post pages, per-tag pages, a tag cloud, an archive, an Atom feed, and a hyde-style left sidebar.  =M-x org-bootstrap-publish= writes =public/=.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/9e022ee][9e022ee]])
- Dev server with rebuild-on-save: =M-x org-bootstrap-publish-serve= starts =python3 -m http.server=, opens the browser, and installs an =after-save-hook= on the source file so saves trigger an incremental rebuild.  =M-x org-bootstrap-publish-stop= kills the server and removes the hook.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/9e022ee][9e022ee]])
- Local build+push deploy: =M-x org-bootstrap-publish-publish= builds, rsyncs the output into a pinned deploy worktree, commits, and pushes.  A =.nojekyll= is written so GitHub Pages serves underscore-prefixed paths.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/9e022ee][9e022ee]])
- Sample GitHub Actions workflow at =workflows/pages.yml= for sites that prefer CI-side builds.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/9e022ee][9e022ee]])
- Client-side article search.  Generates =public/index.json= (one entry per post with title, permalink, summary, tags, section, date; summary stripped of Org markup and truncated to 200 chars).  Ships =assets/search.js= once per site.  Sidebar input shows up to 12 matches with arrow-key navigation and Enter-to-go.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/3c0e450][3c0e450]])
- Asynchronous full build.  =M-x org-bootstrap-publish-async= spawns a child =emacs --batch -Q= subprocess so the editor stays responsive; progress lines stream to the echo area and the full transcript lands in =*obp-build*=.  =org-bootstrap-publish-serve= now uses the async path for its initial build; the server and browser launch from the completion callback.  =org-bootstrap-publish-async-abort= and =org-bootstrap-publish-stop= kill a running child cleanly.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/0836a94][0836a94]])

** Changed

- Feed layout aligned with Hugo's defaults so migrating sites keep their existing RSS subscribers without re-notifying the archive.  Posts now live at =/<section>/<slug>/=, slugs are lowercased, and the Atom feed is mirrored at =index.xml= (with =feed.xml= kept as an alias).  Entry =<id>=s use the full permalink, matching Hugo's =<guid>=.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/d0e9bbe][d0e9bbe]])
- Per-tag feeds at =/tags/<tag>/index.xml= and tag HTML at =/tags/<tag>/index.html= so aggregators like =planet.emacslife.com= keep resolving.  =--feed= gained optional =title= and =rel-path= arguments for per-tag feed identities.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/8ce257e][8ce257e]])
- Site tagline is now configurable via =org-bootstrap-publish-site-tagline=.  ([[https://github.com/captainflasmr/org-bootstrap-publish/commit/5269faa][5269faa]])
