#+title: org-bootstrap-publish
#+author: James Dyer
#+options: toc:nil

Generate a Bootstrap 5 static site from a single Org file.  Each top-level heading becomes a post.  Output is a =public/= folder that can be uploaded to any static host (Cloudflare Pages, Netlify, GitHub Pages, plain S3, etc.).

* Features

- Index page with responsive Bootstrap 5 post cards.
- One HTML page per post, rendered via =ox-html=.
- One HTML page per tag, plus a tag-cloud index.
- Atom feed at =index.xml= (also mirrored to =feed.xml=).
- Hyde-style left sidebar with site title, tagline, nav, and byline.
- Optional highlight.js code-block highlighting (CDN, opt-out).
- Gallery posts: a heading marked =:EXPORT_HUGO_TYPE: gallery= renders a masonry grid of every image found in =static/<section>/= at build time, in addition to the heading body.  Useful for art portfolios and similar collections where the images live next to each other on disk and you want one page that surfaces them all.
- Hugo shortcodes: =\{\{< youtube ID >\}\}=, =\{\{< video src="..." >\}\}=, and =\{\{< figure src="..." caption="..." >\}\}= are recognised both bare and wrapped in =#+begin_export md= / =#+end_export=, and emit a Bootstrap iframe, =<video>=, or =<figure>= in their place.  Lets an ox-hugo source file with embedded shortcodes carry over without manual rewriting.
- Recognises =EXPORT_FILE_NAME= (use =index= for a section-bundle URL), =EXPORT_HUGO_SECTION=, =EXPORT_HUGO_LASTMOD=, =EXPORT_HUGO_TYPE=, and the =:thumbnail= key inside =EXPORT_HUGO_CUSTOM_FRONT_MATTER+= so an ox-hugo source file works unchanged.

* Install

Drop the directory on your =load-path=:

#+begin_src elisp
(add-to-list 'load-path "~/source/repos/org-bootstrap-publish")
(require 'org-bootstrap-publish)
#+end_src

* Usage

Configure once per site, then call =M-x org-bootstrap-publish=:

#+begin_src elisp
(setq org-bootstrap-publish-source-file "~/publish/hugo-unified/README.org"
      org-bootstrap-publish-output-dir  "~/publish/obp-out/public"
      org-bootstrap-publish-site-title  "Emacs Dwelling"
      org-bootstrap-publish-site-tagline "Emacs notes from James Dyer"
      org-bootstrap-publish-site-url    "https://emacs.dyerdwelling.family/"
      org-bootstrap-publish-author      "James Dyer")
#+end_src

Or run it from the command line:

#+begin_src sh
emacs --batch \
  -L ~/source/repos/org-bootstrap-publish \
  -l org-bootstrap-publish \
  --eval "(progn
    (setq org-bootstrap-publish-source-file \"/path/to/README.org\")
    (setq org-bootstrap-publish-output-dir  \"./public\")
    (setq org-bootstrap-publish-site-title  \"My Site\")
    (org-bootstrap-publish))"
#+end_src

* Multiple sites from one Emacs

Every option above is a regular =defcustom=, so the easiest way to publish more than one site is to override them per build instead of =setq='ing them globally.  Pass overrides on the =--batch= command line and the same Emacs build can target whichever site you point it at — the in-Emacs default values still resolve to the =Emacs Dwelling= site, but the batch =--eval= takes precedence.

The example below builds the art portfolio.  It overrides the source file, output dir, URL, title, the static subtree to copy, and the sidebar tag links.  The =EXPORT_HUGO_TYPE: gallery= headings in =art.org= are picked up automatically.

#+begin_src sh
emacs --batch \
  -L ~/source/repos/org-bootstrap-publish \
  -l org-bootstrap-publish \
  --eval "(progn
    (setq org-bootstrap-publish-source-file  \"~/publish/hugo-unified/art.org\")
    (setq org-bootstrap-publish-output-dir   \"~/publish/hugo-unified/public\")
    (setq org-bootstrap-publish-site-title   \"DyerDwelling/art\")
    (setq org-bootstrap-publish-site-tagline \"Welcome to the home of James Dyer wearied traveller\")
    (setq org-bootstrap-publish-site-url     \"https://www.art.dyerdwelling.family/\")
    (setq org-bootstrap-publish-site-path    \"/\")
    (setq org-bootstrap-publish-author       \"James Dyer\")
    (setq org-bootstrap-publish-static-dirs
          '(\"static/art--gallery\" \"static/art--videos\" \"static/art--other\"))
    (setq org-bootstrap-publish-menu-tags
          '((\"gallery\" . \"gallery\")
            (\"videos\"  . \"videos\")
            (\"doodles\" . \"doodle\")
            (\"artrage\" . \"artrage\")
            (\"2024\"    . \"2024\")
            (\"2023\"    . \"2023\")
            (\"2022\"    . \"2022\")))
    (org-bootstrap-publish))"
#+end_src

The interactive =M-x org-bootstrap-publish= still uses whatever your =init.el= sets — typically the emacs blog — so day-to-day editing of =README.org= continues to work without thinking about it.  A second site is one shell command away when you need it.

** Multi-source sites

A single site can pull posts from several Org files at once.  Set =org-bootstrap-publish-source-files= (plural) to the list — it takes precedence over =-source-file= when both are set.  Headings from every file are merged, sorted by =EXPORT_HUGO_LASTMOD=, and treated as if they came from one source.  Useful for an umbrella site that aggregates an art blog, a Linux blog, an Emacs blog, etc.

#+begin_src elisp
(setq org-bootstrap-publish-source-files
      '("~/publish/hugo-unified/blog.org"
        "~/publish/hugo-unified/art.org"
        "~/publish/hugo-unified/linux.org"
        "~/publish/hugo-unified/kate.org"
        "~/publish/hugo-unified/README.org"))
#+end_src

The save-hook rebuild walks all configured files looking for the buffer being saved, so editing any one of them still triggers an incremental rebuild.

The umbrella site usually wants the sidebar to link section landings (=/blog/=, =/photos/=) and specific posts rather than the per-tag links in =org-bootstrap-publish-menu-tags=.  =org-bootstrap-publish-menu-links= takes an alist of =(label . url)= pairs and renders them in the sidebar as raw URLs:

#+begin_src elisp
(setq org-bootstrap-publish-menu-links
      '(("gallery"    . "/art--gallery/")
        ("photos"     . "/photos/")
        ("blog"       . "/blog/")
        ("emacs"      . "/tags/emacs/")
        ("about"      . "/blog/posts--about-me/")))
#+end_src

Section landings (=/blog/=, =/photos/=, =/photos/2003/=, etc.) are generated automatically — one listing per distinct =EXPORT_HUGO_SECTION= root — unless a heading with =EXPORT_FILE_NAME: index= already owns that URL.

** Selective static copy

=org-bootstrap-publish-static-dirs= controls which subtrees of the source file's directory are copied into =public/=.  The default is the whole =static/= folder, which is fine when one folder serves one site.  Set it to a list of nested paths to copy only what a given site needs:

#+begin_src elisp
;; Art site: skip emacs/, kate/, photos/, etc.
(setq org-bootstrap-publish-static-dirs
      '("static/art--gallery" "static/art--videos" "static/art--other"))
#+end_src

The destination directory tree is created automatically, so nested entries work without any further setup.

* Customization

| Variable                                      | Meaning                                            |
|-----------------------------------------------+----------------------------------------------------|
| =org-bootstrap-publish-source-file=           | The Org file to publish.                           |
| =org-bootstrap-publish-source-files=          | List of Org files to publish (overrides above).    |
| =org-bootstrap-publish-menu-links=            | Sidebar =(label . url)= alist for arbitrary links. |
| =org-bootstrap-publish-output-dir=            | Output directory (default =public/= under pwd).    |
| =org-bootstrap-publish-site-title=            | Sidebar title.                                     |
| =org-bootstrap-publish-site-tagline=          | Sidebar sub-title.                                 |
| =org-bootstrap-publish-site-url=              | Canonical URL, used in the Atom feed.              |
| =org-bootstrap-publish-site-path=             | URL prefix (=/= for root, =/repo/= for subpath).   |
| =org-bootstrap-publish-author=                | Author name.                                       |
| =org-bootstrap-publish-posts-per-page=        | How many posts appear on the index page.           |
| =org-bootstrap-publish-exclude-tags=          | Tags that skip a heading (default =("noexport")=). |
| =org-bootstrap-publish-static-dirs=           | Dirs copied verbatim next to the source file.      |
| =org-bootstrap-publish-bootstrap-css= / =-js= | Bootstrap 5 CDN URLs (override to vendor).         |
| =org-bootstrap-publish-highlight-css= / =-js= | highlight.js CDN URLs (nil to disable).            |

* Post properties

The package reads these properties from each heading's drawer:

| Property                           | Use                                                            |
|------------------------------------+----------------------------------------------------------------|
| =EXPORT_FILE_NAME=                 | Post slug (lowercased; =index= → URL is =/<section>/=).        |
| =EXPORT_HUGO_SECTION=              | URL prefix.  Posts live at =/<section>/<slug>/=.               |
| =EXPORT_HUGO_LASTMOD=              | Post timestamp.  Parses =<YYYY-MM-DD HH:MM>=.                  |
| =EXPORT_HUGO_TYPE=                 | =gallery= renders a masonry grid from =static/<section>/=.     |
| =EXPORT_HUGO_CUSTOM_FRONT_MATTER+= | =:thumbnail /emacs/foo.jpg= → card image.                      |

Heading tags become tag pages.  The =noexport= tag skips a heading.

* Gallery posts

A gallery is just a regular post with two extra drawer entries:

#+begin_src org
,* Gallery -> Portraits :portrait:gallery:artrage:
:PROPERTIES:
:EXPORT_FILE_NAME: index
:EXPORT_HUGO_SECTION: art--gallery/portraits
:EXPORT_HUGO_TYPE: gallery
:EXPORT_HUGO_LASTMOD: 2023-03-05
:EXPORT_HUGO_CUSTOM_FRONT_MATTER+: :thumbnail /art--gallery/gallery--portraits.jpg
:END:
A short paragraph that appears above the grid.

#+hugo: more
#+end_src

At build time the package walks =static/art--gallery/portraits/= relative to the source file, collects every =.jpg/.png/.gif/.webp/.tiff/.bmp=, sorts them by filename descending (so date-prefixed files come out newest-first), and emits a Bootstrap masonry grid pointing at each image.  The =index= slug collapses the URL to =/art--gallery/portraits/= to match Hugo's content-bundle convention.

For this to work, =static/art--gallery= (or whichever subtree holds the gallery images) must be in =org-bootstrap-publish-static-dirs= so it gets copied to the output.  See "Selective static copy" below.

* Preview

** Inside Emacs

=M-x org-bootstrap-publish-serve= builds, opens a browser at =http://localhost:8080=, and installs an =after-save-hook= so editing the source file rebuilds just that post plus the listings (~1 second).  =M-x org-bootstrap-publish-stop= shuts the server down again.

** From a shell

After a batch build, serve the =public/= folder directly:

#+begin_src sh
cd public && python3 -m http.server 8080
#+end_src

Then walk a few representative URLs to spot-check the build before deploying — at minimum the index, a regular post, the archive, the tags index, and (if relevant) a gallery page:

#+begin_src
http://localhost:8080/
http://localhost:8080/posts.html
http://localhost:8080/tags.html
http://localhost:8080/<section>/<slug>/             # a regular post
http://localhost:8080/art--gallery/portraits/       # a gallery page
http://localhost:8080/index.xml                     # Atom feed
#+end_src

* Deploy

- Cloudflare Pages: =wrangler pages deploy public --project-name=<name>=
- Netlify:          drag-drop =public/=, or =netlify deploy --prod --dir public=
- GitHub Pages:     see =workflows/pages.yml= for a CI-side build, or
                    commit =public/= on a =gh-pages= branch
- Local push:       =M-x org-bootstrap-publish-publish= rsyncs into
                    =org-bootstrap-publish-deploy-dir=, commits, pushes
